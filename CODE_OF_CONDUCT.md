# Code Of Conduct

FulFrame should be a practical, respectful place to build reusable Flutter app
foundation work.

## Expected Behavior

- Be direct, specific, and respectful in technical discussion.
- Assume contributors are trying to improve the project, then ask clarifying
  questions when intent or scope is unclear.
- Keep feedback focused on code, docs, architecture, tests, and project goals.
- Respect maintainer decisions about MVP scope, architecture boundaries, and
  release timing.
- Help new contributors understand the GDD, epic, and story workflow.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discriminatory comments or exclusionary behavior.
- Publishing private information without permission.
- Repeatedly derailing issues or pull requests after maintainers have clarified
  scope.
- Spam, malicious links, or knowingly harmful contributions.

## Reporting

For now, report conduct concerns by opening a GitHub issue if public discussion
is appropriate. If the concern should not be public, contact the repository
maintainer directly through their GitHub profile.

Do not include private personal details in a public issue.

## Enforcement

Maintainers may edit or remove comments, close issues or pull requests, block
users from the repository, or take other reasonable action to protect the
project and its contributors.

This document may become more formal as the project grows.
